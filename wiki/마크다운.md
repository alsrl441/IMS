# 1. 개요

마크다운(Markdown)은 마크업 언어의 일종으로, 복잡한 태그 없이 일반 텍스트만으로 문서의 구조와 서식을 간편하게 작성할 수 있게 만든 '경량 마크업 언어'입니다.
마크다운을 활용하면 복잡한 HTML 없이도 가독성 높은 문서를 빠르고 간편하게 작성할 수 있습니다.

---

# 2. 문법

## 2-1. 문단 제목

`#`(해시) 기호를 사용해 문서의 섹션을 나눌 수 있습니다.
`#`의 개수가 많아질수록 하위 제목이 되며, 6단계까지 할 수 있습니다.

# 1단계 제목
`# 1단계 제목`

## 2단계 제목
`## 2단계 제목`

### 3단계 제목
`### 3단계 제목`

#### 4단계 제목
`#### 4단계 제목`

##### 5단계 제목
`##### 5단계 제목`

###### 6단계 제목
`###### 6단계 제목`

---

## 2-2. 목록

정보를 나열할 때 사용하며, 자동으로 들여쓰기가 적용됩니다.

### 순서 없는 목록
줄 시작 부분에 `-`(하이픈) 또는 `*`(애스터리스크), `+`(플러스)를 쓰고 한 칸 띄우고 작성합니다. 하위 항목은 공백 4칸 후 작성합니다

- 항목 1
* 항목 2
+ 항목 3
    - 1차 하위 항목 1
    * 1차 하위 항목 2
    + 1차 하위 항목 2
        - 2차 하위 항목 1
        * 2차 하위 항목 2
        + 2차 하위 항목 2

### 순서 있는 목록
줄 시작 부분에 숫자를 입력합니다. 하위 항목은 `-`으로 이어서 작성합니다.

1. 첫 번째 순서
2. 두 번째 순서
	2-1. 두 번째 순서 하위 첫 번째 순서
	2-2. 두 번째 순서 하위 두 번째 순서
		2-2-1. 두 번째 순서 하위 두 번째 순서 하위 첫 번째 순서
		2-2-2. 두 번째 순서 하위 두 번째 순서 하위 두 번째 순서
	2-3. 두 번째 순서 하위 세 번째 순서
	2-4. 두 번째 순서 하위 세 번째 순서
3. 세 번째 순서

> `#` 뒤에 순서 있는 목록을 붙이면 목차(TOC)에 자동으로 등록되어 클릭 시 해당 위치로 이동할 수 있습니다.

## 2-3. 글자 모양

글자 모양을 변경하여 특정 단어를 강조할 수 있습니다.

### **굵게**
`**텍스트**`

### *기울임*
`*텍스트*`

### ~취소선~
`~텍스트~`

### _밑줄_
`_텍스트_`

> ** * ~ _여러 개의 효과를 중첩 적용할 수 있습니다._ ~ * **

## 2-4. 코드

문서 내에 코드를 입력할 필요가 있을 때 다음과 같이 사용합니다.

한 줄 코드는 `\``(백틱) 1개로 감싸서 사용합니다.
`\`한 줄 코드입니다.\``

여러 줄 코드는 `\`` 3개로 감싸서 사용합니다.
```
\`\`\` javascript
let string = "여러 줄 코드입니다.";
console.log(string);
\`\`\`
```

> 코드 내에서 `\`` 또는 `\\`(백슬래시/원화 기호)을 사용할 때는 앞에 `\\`를 붙여서 사용합니다.

---

## 2-5. 인용구

참고 문헌이나 강조하고 싶은 문장을 작성할 때 줄 시작 부분에 `>`(그래이터 댄)를 입력하여 사용합니다. 

> "이곳에 인용할 내용을 작성합니다. 좌측 파란색 테두리와 회색 배경색이 자동으로 적용됩니다."

---

## 2-6. 구분선
내용의 흐름을 끊거나 주제를 바꿀 때 한 줄에 `---`(하이픈 3개)를 입력하여 사용합니다.
---

---

## 2-7. 외부 링크

외부 사이트를 연결할 수 있습니다.
`[]`(대괄호) 안에 표시할 이름과 `()`(소괄호) 안에 연결할 주소를 입력하여 작성합니다.

`[표시명](주소)`
[구글 바로가기](https://google.com/)

---

## 2-8. 내부 링크

Wiki 내의 다른 문서나 특정 위치로 이동할 수 있습니다.

- **기본 연결**: `[[문서명]]` (예: `[[마크다운]]` [[마크다운]])
- **표시 이름 변경**: `[[문서명|표시명]]` (예: `[[마크다운|마크다운 가이드 문서]]` [[마크다운|마크다운 가이드 문서]])
- **현재 문서 내 특정 위치로 이동**: `[[#목차명|표시명]]` (예: `[[#2-8. 내부 링크|내부 링크]]` [[#2-8. 내부 링크|내부 링크]])
- **다른 문서의 특정 위치로 이동**: `[[문서명#목차명|표시명]]` (예: `[[마크다운#2-8. 내부 링크|내부 링크]]` [[마크다운#2-8. 내부 링크|내부 링크]])

> 연결하려는 문서가 없으면 클릭 시 새 문서 작성을 제안합니다.

---

## 2-9. 이미지 삽입

외부 이미지를 출력할 수 있습니다.
`!`를 입력한 다음 `[]` 안에 이미지 설명과 `()`안에 이미지 주소 또는 저장 경로를 입력하여 작성합니다.

`![예시 이미지](./image.jpg)` ![예시 이미지](./image.jpg)

## 2-10. 	표

`|`(수직선)과 `---`으로 테이블 구조를 만들어 작성합니다.

```
| 구분 | col1 | col2 |
|---|---|---|
| row1 | val1 | val2 | 
| row2 | val3 | val4 |
| row2 | val5 | val6 |
```

| 구분 | col1 | col2 |
|---|---|---|
| row1 | val1 | val2 | 
| row2 | val3 | val4 |
| row2 | val5 | val6 |

# 3. 주의사항

마크다운은 표준화된 단일 규격이 존재하지 않고, 서비스마다 고유의 확장 문법을 사용합니다.
이를 '마크다운의 파편화'라고 합니다.
본 문서는 이용 편의를 위해 독자적인 문법을 포함하고 있습니다. 따라서 상기 작성된 문법의 일부 서식은 외부 플랫폼에서는 의도와 다르게 보일 수 있음을 알립니다.
