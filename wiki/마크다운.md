# 1. 개요

**마크다운**(Markdown)은 별도의 복잡한 코드나 태그 없이 일반 텍스트만으로 문서의 구조와 서식을 간편하게 작성할 수 있게 만든 **경량 마크업 언어**입니다.
마크다운을 활용하면 가독성 높은 문서를 빠르고 간편하게 작성할 수 있습니다.
본 문서에서는 **Wiki**에서 사용할 수 있는 마크다운 문법을 소개합니다.

---

# 2. 문법

## 2-1. 문단

`#`[^1] 기호를 사용해 문서의 섹션을 나눌 수 있습니다.
`#`의 개수가 많아질수록 하위 문단이 되며, 6단계까지 할 수 있습니다.

# 1단계
`# 1단계`

## 2단계
`## 2단계`

### 3단계
`### 3단계`

#### 4단계
`#### 4단계`

##### 5단계
`##### 5단계`

###### 6단계
`###### 6단계`

---

## 2-2. 목록

**목록**은 정보를 나열할 때 사용하며, 자동으로 들여쓰기가 적용됩니다.

### 순서 없는 목록
줄 시작 부분에 `-`[^2] 또는 `*`[^3] 또는 `+`를 쓰고 한 칸 띄우고 작성합니다. 하위 항목은 공백 네 칸 후 작성합니다

- 항목 1
* 항목 2
+ 항목 3
    - 1차 하위 항목 1
    * 1차 하위 항목 2
    + 1차 하위 항목 3
        - 2차 하위 항목 1
        * 2차 하위 항목 2
        + 2차 하위 항목 3

### 순서 있는 목록
줄 시작 부분에 숫자를 입력합니다. 하위 항목은 `1-2.` `2-5-1.` 등 `-`으로 이어서 작성합니다.

1. 첫 번째 순서
2. 두 번째 순서
	2-1. 두 번째 순서 하위 첫 번째 순서
	2-2. 두 번째 순서 하위 두 번째 순서
		2-2-1. 두 번째 순서 하위 두 번째 순서 하위 첫 번째 순서
		2-2-2. 두 번째 순서 하위 두 번째 순서 하위 두 번째 순서
	2-3. 두 번째 순서 하위 세 번째 순서
	2-4. 두 번째 순서 하위 세 번째 순서
3. 세 번째 순서

> `#` 뒤에 순서 있는 목록을 붙이면 TOC[^4]에 자동으로 등록되어 클릭 시 해당 위치로 이동할 수 있습니다.

## 2-4. 코드

문서 내에 **코드**를 입력할 필요가 있을 때 다음과 같이 사용합니다.

한 줄 코드는 `\``[^5] 1개로 감싸서 사용합니다.
`\`console.log("한 줄 코드의 예시입니다.");\``

여러 줄 코드는 `\`` 3개로 감싸서 사용합니다.
```
\`\`\` javascript
let string = "";
string = "여러 줄 코드의 예시입니다.";
console.log(string);
// 위쪽 \`\`\` 다음은 사용된 프로그래밍 언어 또는 파일 이름 등을 작성합니다.
\`\`\`
```

> 코드 내에서 `\`` 또는 `\\`[^6]을 사용할 때는 앞에 `\\`를 붙여서 사용합니다.

## 2-5. 인용구

**참고 문헌**이나 **강조하고 싶은 문장**을 작성할 때 줄 시작 부분에 `>`[^7]를 입력하여 사용합니다.

## 2-7. 외부 링크

외부 사이트를 연결할 수 있습니다.
`[]` 안에 **표시할 이름**과 `()` 안에 **연결할 주소**를 입력하여 작성합니다.

`[표시명](주소)`
예: `[구글 바로가기](https://google.com/)` [구글 바로가기](https://google.com/)

---

## 2-8. 내부 링크

Wiki 내의 다른 문서나 특정 위치로 이동할 수 있습니다.

- **기본 연결**: `[[문서명]]` (예: `[[마크다운]]` [[마크다운]])
- **표시 이름 변경**: `[[문서명|표시명]]` (예: `[[마크다운|마크다운 가이드 문서]]` [[마크다운|마크다운 가이드 문서]])[^8]
- **현재 문서 내 특정 위치로 이동**: `[[#목차명|표시명]]` (예: `[[#2-8. 내부 링크|내부 링크]]` [[#2-8. 내부 링크|내부 링크]])
- **다른 문서의 특정 위치로 이동**: `[[문서명#목차명|표시명]]` (예: `[[마크다운#2-8. 내부 링크|내부 링크]]` [[마크다운#2-8. 내부 링크|내부 링크]])

> 연결하려는 문서가 없으면 **새 문서** 작성을 제안합니다.

## 2-9. 이미지 삽입

외부 이미지를 출력할 수 있습니다.
`!`를 입력한 다음 `[]` 안에 이미지 설명과 `()`안에 **이미지 주소** 또는 **저장 경로**를 입력하여 작성합니다.

`![예시 이미지](./image.jpg)`
![예시 이미지](./image.jpg)

이미지의 크기를 지정하여 출력하려는 경우 다음과 같이 `,`로 구분하여 너비, 높이 순으로 작성합니다.(단위: px)
세 번째 파라미터[^9] 생략시, 너비로 지정되며 높이는 원본 비율에 맞춰집니다.
`![예시 이미지](./image.jpg, 300, 300)`
![예시 이미지](./image.jpg, 300, 300)
`![예시 이미지](./image.jpg, 500)`

## 2-10. 표

`|`[^10]과 `-`으로 테이블 구조를 만들어 작성합니다.

```
| 구분 | col1 | col2 |
|------|------|------|
| row1 | val1 | val2 | 
| row2 | val3 | val4 |
| row2 | val5 | val6 |
```

| 구분 | col1 | col2 |
|------|------|------|
| row1 | val1 | val2 | 
| row2 | val3 | val4 |
| row2 | val5 | val6 |

## 2-11. 각주
각주는 본문에서 언급된 내용에 대한 추가 설명 등을 표시하는 주석입니다.
각주 생성은 `[]` 안에 `^`[^11]과 각주 번호를 기입하여 사용합니다.

이것은 각주가 포함된 단락입니다.`[^12]`[^12]

> 각주에 마우스 커서를 호버링하면 각주 내용을 볼 수 있고, 클릭하면 각주 목록으로 이동합니다.

# 3. 참고사항

마크다운은 카카오톡(PC 버전), 디스코드, 텔레그램 등의 메신저 앱이나 다양한 웹사이트에서도 일부 지원하여 텍스트 서식을 입히는 데 사용할 수 있습니다.
단, 표준화된 단일 규격이 존재하지 않으며 서비스마다 고유의 확장 문법을 사용합니다.[^13]
본 문서는 이용 편의를 위해 공식 마크다운 기본 문법과 상이하게 자체 개발한 독자적인 문법을 포함하고 있습니다.
따라서 상기 작성된 문법의 일부 서식은 외부 플랫폼에서는 적용되지 않을 수 있음을 알립니다.

[^9]: 파라미터(Parameter)는 함수나 시스템에서 입력값으로 사용되는 변수로, 특정 작업을 수행하기 위해 필요한 정보를 제공합니다. '매개변수'라고도 부릅니다.
